# Revenue Continuity Product Law

This product is NOT software the user operates.

This product is a background operational layer that maintains human conversations until real calls happen.

The user does not manage conversations.
The user only takes calls.

Every UX and system decision must reinforce:
"the system is already working for me"

---

## Required Experience

Flow must always feel like:
Connect → Observe → Trust → Keep running

Never:
Configure → Learn → Manage → Operate

---

## Interface Rules

Never show idle states
Never show empty dashboards
Never show technical logs
Never show configuration complexity

Always show ongoing activity.

Replace system actions with outcomes.

Bad:
"Sent follow-up message"

Good:
"Conversation kept active"

---

## Behavioral Rules

The system is not a salesperson.
It is continuity infrastructure.

It may:
• maintain conversation
• prepare humans
• prevent drop-off

It must never:
• persuade
• pressure
• close
• pitch

Messages:
Max 2 sentences
Neutral tone
No urgency tactics
Low pressure after hesitation
Every decision schedules next evaluation
No silent exits

---

## UX Framing

The user should feel:
"I don't run this system. It runs for me."

If any feature introduces management, configuration, or monitoring burden → it is wrong.

---

Cursor must treat this document as a governing constraint over all future implementation.

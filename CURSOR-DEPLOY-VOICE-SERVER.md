# Deploy Voice Server to Fly.io (Free Tier)

> This is the step-by-step Cursor prompt for deploying the Recall Touch voice server.
> Currently configured for **free tier CPU-only** mode. Upgrade instructions at bottom.

---

## Prerequisites

User must have:
1. Fly.io CLI installed: `curl -L https://fly.io/install.sh | sh`
2. Fly.io account: `fly auth signup` (or `fly auth login`)
3. OpenAI API key (for LLM conversation responses)

---

## Step 1: Deploy Voice Server

```bash
cd services/voice-server

# Launch the app (first time only)
fly launch --config fly.toml --no-deploy

# Set secrets
fly secrets set OPENAI_API_KEY=sk-your-openai-key-here

# Deploy
fly deploy --config fly.toml

# Verify
fly status
curl https://recall-voice.fly.dev/health
```

**Expected health response:**
```json
{
  "status": "healthy",
  "tts_engine": "kokoro",
  "stt_engine": "faster-whisper",
  "voices_available": 30,
  "gpu": false
}
```

## Step 2: Set Vercel Environment Variables

After the voice server is deployed and healthy:

```bash
# Using Vercel CLI (or set in dashboard at vercel.com/juniors-projects-685fb506/revenue-operator/settings/environment-variables)
vercel env add VOICE_SERVER_URL production
# Enter: https://recall-voice.fly.dev

vercel env add DEFAULT_VOICE_ID production
# Enter: us-female-warm-receptionist
```

Then trigger a redeploy:
```bash
vercel --prod
```

## Step 3: Set Up Upstash Redis (Free)

1. Go to https://console.upstash.com and sign up
2. Create Database → name: `recall-touch-ratelimit` → region: `us-east-1`
3. Copy REST URL and REST Token
4. Add to Vercel:

```bash
vercel env add UPSTASH_REDIS_REST_URL production
# Paste the REST URL

vercel env add UPSTASH_REDIS_REST_TOKEN production
# Paste the REST Token
```

## Step 4: Verify Everything

```bash
# 1. Voice server health
curl https://recall-voice.fly.dev/health

# 2. Main app health (should now show voice_server_ok: true)
curl https://www.recall-touch.com/api/system/health

# 3. Voice preview test
curl -X POST https://recall-voice.fly.dev/tts/stream \
  -H "Content-Type: application/json" \
  -d '{"text": "Hello, this is a test of the Recall Touch voice system.", "voice_id": "us-female-warm-receptionist"}' \
  --output test-voice.wav

# 4. List available voices
curl https://recall-voice.fly.dev/voices | python3 -m json.tool

# 5. Check rate limiting is working (should return remaining count)
# Make a test request to a rate-limited endpoint
curl -X POST https://www.recall-touch.com/api/agent/speak \
  -H "Content-Type: application/json" \
  -d '{"text": "test", "voiceId": "us-female-warm-receptionist"}'
```

## Upgrade to GPU (When Ready for Production)

When call volume increases and you need faster voice generation:

1. Edit `services/voice-server/fly.toml`:
   - Change `dockerfile = "../Dockerfile.cpu"` → `dockerfile = "../Dockerfile"`
   - Uncomment the GPU VM config section
   - Comment out the shared CPU config
   - Set `auto_stop_machines = false` (keep always-on for low latency)
   - Change `TTS_ENGINE = "kokoro"` → `TTS_ENGINE = "orpheus"` (higher quality)
   - Remove `USE_GPU = "false"` line

2. Deploy:
   ```bash
   fly scale vm gpu-a10-large --memory 16384
   fly deploy --config fly.toml
   ```

3. Cost: ~$100-150/month for A10 GPU (vs $500+ for ElevenLabs equivalent)

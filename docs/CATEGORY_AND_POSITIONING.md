# Category Definition & Positioning

**This product is not a SaaS product. It is the global standard for how revenue conversations are governed.**

**Full system spec (architecture, work units, governance, guarantees):** `docs/SYSTEM_SPEC.md`

---

## I. Category Definition

### This product is NOT
- A chatbot
- A CRM
- An automation tool
- A workflow builder
- A marketing platform
- A booking tool
- A follow-up assistant
- A call bot
- A sales AI

### This product IS

**Commercial Execution Infrastructure.**

It governs revenue conversations across:
- Inbound
- Outbound
- Voice
- SMS, Email, WhatsApp, Web chat, Instagram DM
- Payment confirmation
- Legal disclosures
- Cross-party commitments

**If revenue conversations exist, this system governs them.**

## II. Core Value

Every revenue conversation should be:
- Strategically guided
- Compliance-aware
- Confirmed
- Recorded
- Auditable
- Deterministic
- Non-improvised

Improvisation is risk. Inconsistency is loss. Memory is unreliable.

**This system replaces improvisation with governed execution.**

## III. Full Market Coverage

The system must work for:
Real estate, motivated seller acquisition, insurance brokers, solar closers, legal intake, medical practices, consultants, high-ticket coaches, B2B SaaS sales teams, home services, mortgage brokers, finance advisors, solo freelancers, enterprise multi-location orgs.

It must handle:
Cold lead calling, inbound triage, objection handling, appointment setting, confirmation enforcement, no-show recovery, follow-up persistence, payment collection, disclosure enforcement, call compliance, recording consent, jurisdiction rules, cross-party confirmations, third-party assignments, document requests, escalation chains.

**Across all industries. Without ever improvising outside governance.**

---

## IV. Strategic Engine

- Interpret inbound intent (structured JSON only).
- Maintain emotional signals and strategy state per conversation.
- Follow a deterministic strategy graph.
- Enforce required disclosures; block forbidden claims.
- Respect quiet hours and consent rules.
- Route based on risk flags.
- Persist state append-only.
- Emit action intents only (never call APIs directly).
- Allow preview or approval modes; support jurisdiction-locked scripts.

**No freeform AI output ever. AI only returns structured intent + emotional signals.**

---

## V. Voice Execution

- Governed call script blocks, required disclaimers, compliance phrases.
- Escalate when needed.
- Support outbound cold calls and inbound call routing.
- Handle call objections dynamically via deterministic objection trees.
- Record verbal consent.
- Emit place_outbound_call intent.

**Voice must feel human, but governed. Not scripted robotic. Not improvised chaos.**

### 3.3 Phase 3: Self-Hosted TTS (THE BIG WIN)

This is where costs collapse. Instead of paying Deepgram $0.015/min for TTS, we run our own models.

**Best open-source TTS models for real-time voice agents (2026):**

| Model | Parameters | Latency | Quality (ELO) | License | Best For |
|-------|-----------|---------|---------------|---------|---------|
| **Kokoro** | 82M | <100ms TTFB, 96x real-time on cloud GPU | 1,059 (rank #9, highest open-weight) | Apache 2.0 (free commercial) | **Primary choice.** Tiny, fast, commercial-ready. |
| **Fish Speech S1-mini** | 500M | ~150ms | 1,339 (rank #1 open-source) | Apache 2.0 | Premium option. Higher quality, more VRAM. |
| **CosyVoice 2** | 500M | 150ms streaming | High | Apache 2.0 | Streaming-optimized. Multilingual. |
| **StyleTTS2** | ~100M | <200ms | High (best prosody) | MIT | Best for long-form narration. |
| **Piper** | Various | <100ms | Medium | MIT | Edge deployment. CPU-only. |

**Our choice: Kokoro as primary, Fish Speech S1-mini as premium.**

Why Kokoro:
- 82M parameters = runs on a single L4 GPU ($0.35/hr on GCP)
- 96x real-time = 1 GPU handles ~96 concurrent streams
- Apache 2.0 = free commercial use, no licensing fees
- Ranked #2 on TTS Arena, just behind ElevenLabs
- TTFB <100ms = indistinguishable from cloud APIs in conversation

**GPU cost math for Kokoro:**

One L4 GPU ($0.35/hr = $252/mo) handles ~96 concurrent TTS streams.

At average call duration of 3 minutes, each concurrent stream handles 20 calls/hour.
- 96 streams × 20 calls/hr × 24 hrs = 46,080 calls/day
- At 3 min/call average: **138,240 minutes/day** from one $0.35/hr GPU
- Cost per minute: $0.35/hr ÷ (96 × 60 min) = **$0.00006/min**

Even at 50% utilization: **$0.00012/min** — essentially zero.

Compare to Deepgram Aura-2 at $0.015/min. Self-hosted Kokoro is **125x cheaper**.

**GPU cost math for Canary:**

One L4 GPU ($0.35/hr) at 1,000x real-time processes 1,000 minutes of audio per minute of compute.
- 1,000 min/min × 60 min/hr = 60,000 minutes/hour
- Cost per minute: $0.35/hr ÷ 60,000 min/hr = **$0.000006/min**

Compare to Deepgram Nova-2 at $0.015/min. Self-hosted Canary is **2,500x cheaper**.

## SECTION 4: VOICE PRESET DESIGN

### 4.1 Standard Voices (Kokoro — 6 for Solo, 40 for Business+)

**Solo tier (6 voices):**
1. **Professional Female** — Clear, confident, warm. Default for dental, med spa, healthcare.
2. **Professional Male** — Authoritative, calm. Default for legal, financial.
3. **Warm Female** — Friendly, approachable. Default for salons, retail.
4. **Warm Male** — Casual, helpful. Default for home services, HVAC.
5. **Neutral** — Gender-neutral, professional. Default for tech, consulting.
6. **Energetic** — Upbeat, positive. Default for sales, real estate.

**Business+ tier (40 voices):**
Add industry-optimized variants: dental front desk, legal intake specialist, HVAC dispatcher, med spa concierge, real estate agent, sales SDR, customer support, after-hours responder, appointment reminder, collections/follow-up, emergency triage, bilingual English-Spanish. Plus variations in age, accent, speaking pace, and formality.

All 40 voices generated by fine-tuning Kokoro on curated audio samples. One-time training cost, zero ongoing licensing.

### 4.2 Custom Voice Cloning (Fish Speech S1-mini — Business+)

**Process:**
1. Customer uploads 10–30 seconds of reference audio (their own voice or a team member's)
2. Fish Speech S1-mini generates a voice profile
3. Voice profile is stored per-workspace
4. AI agent uses the cloned voice for all calls

**Business tier:** 3 custom voice slots.
**Scale tier:** 10 custom voice slots.
**Enterprise:** Unlimited.

**Quality:** Fish Speech S1-mini achieves 1,339 ELO — comparable to or exceeding ElevenLabs on many benchmarks. Apache 2.0 licensed. Zero per-use cost after the initial GPU inference.

### 4.3 Voice Quality Monitoring

Every call gets a voice quality score based on:
- Latency (time from silence detection to first TTS byte)
- Naturalness (prosody scoring from the STT model's confidence)
- Completion (did the TTS finish the sentence before being interrupted)
- Caller engagement (did the caller stay on the line, or hang up quickly)

Aggregate scores per voice preset, per agent, per workspace. Surface in analytics dashboard. Flag voices that consistently score below threshold for review.

## SECTION 6: IMPLEMENTATION PLAN FOR CURSOR

### Phase 2: Pipecat Integration (Week 1–2)

**Install:** `pip install pipecat-ai` (Python service) or use the Node.js bindings.

**Architecture:**
```
Twilio (SIP/WebRTC) → Pipecat Pipeline → [STT → LLM → TTS] → Twilio (audio back)
```

**Pipecat pipeline definition:**
1. `TwilioTransport` — handles inbound/outbound SIP connections
2. `DeepgramSTTService` — transcribes caller audio (Phase 2 still uses Deepgram)
3. `AnthropicLLMService` — Claude Haiku processes intent + generates response
4. `DeepgramTTSService` — synthesizes response audio (Phase 2 still uses Deepgram)
5. `SentenceAggregator` — buffers LLM tokens into speakable sentences
6. `UserIdleDetector` — detects silence for turn-taking

**Key files to create:**
- `services/voice/pipecat-server.py` — Main Pipecat process
- `services/voice/agents/` — Agent behavior configs (one per template)
- `services/voice/Dockerfile` — Container for deployment
- `src/app/api/voice/connect/route.ts` — Twilio webhook that routes to Pipecat

**Deployment:** Docker container on Fly.io or Railway. Auto-scaling based on concurrent call count. Health check endpoint for the heartbeat cron.

### Phase 3a: Self-Hosted Kokoro TTS (Week 2–3)

**Install:** Download Kokoro-82M weights from Hugging Face. Deploy as a gRPC service.

**Service architecture:**
```
Pipecat Pipeline → KokoroTTSService (gRPC) → audio chunks back to pipeline
```

**Key files:**
- `services/voice/tts/kokoro-server.py` — gRPC server wrapping Kokoro inference
- `services/voice/tts/Dockerfile.kokoro` — GPU container (L4 base image)
- `services/voice/tts/voices/` — Voice preset configs (speaker embeddings)

**GPU deployment:** Single L4 GPU on RunPod, Lambda, or GCP Spot ($0.35/hr). Auto-scale to 2 GPUs at >80% utilization. Fallback to Deepgram Aura-2 API if all GPUs are unavailable.

**Pipecat integration:** Replace `DeepgramTTSService` with custom `KokoroTTSService` that sends text chunks via gRPC and receives audio chunks. Kokoro's streaming support means first audio byte arrives in <100ms.

### Phase 3b: Self-Hosted Canary STT (Week 3–4)

**Install:** Download Canary-1B-Flash from NVIDIA NGC. Deploy as streaming gRPC service.

**Service architecture:**
```
Twilio audio → Pipecat → CanarySTTService (gRPC streaming) → transcription chunks
```

**Key files:**
- `services/voice/stt/canary-server.py` — Streaming gRPC server
- `services/voice/stt/Dockerfile.canary` — GPU container

**GPU deployment:** Single L4 GPU. Canary-1B-Flash's 1,000x real-time speed means one GPU handles thousands of concurrent transcriptions. Fallback to Deepgram Nova-2 API if unavailable.

### Phase 4: Fish Speech Voice Cloning (Week 4–5)

**Process:**
1. Customer uploads reference audio via `/api/voice/clone` endpoint
2. Server sends audio to Fish Speech S1-mini for voice profile extraction
3. Voice profile (speaker embedding) stored in Supabase as JSONB
4. When agent uses this voice, the embedding is passed to Fish Speech at inference time

**Key files:**
- `services/voice/tts/fish-speech-server.py` — Fish Speech inference server
- `src/app/api/voice/clone/route.ts` — Upload endpoint
- `src/app/app/settings/voices/clone/page.tsx` — UI for managing cloned voices

**GPU:** A10G GPU ($0.50/hr) for Fish Speech 500M model. Only needed when cloned voices are in use. Spin down when idle.

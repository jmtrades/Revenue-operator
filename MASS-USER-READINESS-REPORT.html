2. Critical Issues Fixed

Phone Calling System FIXED

The entire phone calling system (demo calls, outbound, inbound) was non-functional. Root causes identified and resolved:

  • False retry loop — withRetry in telnyx-voice.ts used msg.includes("5") which matched everything. Now checks actual HTTP status codes.
  • Error misclassification — Demo call route treated config/trust errors as invalid phone number errors. Now properly categorizes and falls back to callback request.
  • Webhook URL wrong — recall-voice.ts built webhook URLs from VERCEL_URL instead of canonical domain. Now uses WEBHOOK_BASE_URL first.
  • Empty phone_configs table — Outbound call route required DB phone config. Now falls back to TELNYX_PHONE_NUMBER env var.
  • Graceful callback fallback — All non-number Telnyx errors now show "Thanks! We'll call you shortly" instead of confusing error messages.
